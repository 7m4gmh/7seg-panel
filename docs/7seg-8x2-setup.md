# 7セグLEDパネル（横8桁×縦2桁）設定ガイド

このドキュメントは、横8桁×縦2桁（合計16桁）の7セグLEDパネルを動作させるための設定手順・配線例・config.json のサンプル・テスト方法をまとめたものです。

## 概要
- パネル構成: 横8桁 × 縦2桁（合計16桁）
- 表示方式: ドライバ（MAX7219, HT16K33 等）を使うか、GPIOによるマルチプレックスで直接駆動するか

## 基本ルール
- セグメントビット順は統一する（例: `a,b,c,d,e,f,g,dp` を bit0..bit7 に割当）
- 桁インデックスは row-major（上段左→右: 0..7、下段左→右: 8..15）を推奨

## ハードウェア選択の指針
- MAX7219 (SPI): 複数デバイスをチェーン可能。扱いやすく小規模7セグに最適。
- HT16K33 (I2C): I2Cバスで複数アドレスを扱える。
- GPIO マルチプレックス: 外部トランジスタ等で16桁のコモン駆動が必要。制御は高頻度スキャン（例: 500–1000Hz）。

## 配線要点（代表例）

### MAX7219（SPI）
- `DIN` → MCU `MOSI`
- `CLK` → MCU `SCLK`
- `CS`  → MCU `CE`（/CS）
- VCC/GND を共通に。MCUが3.3VでMAX7219が5Vの場合はレベル変換が必要。

### HT16K33（I2C）
- `SDA`, `SCL` を I2C バスへ接続（プルアップ抵抗を確認）
- 電源・GNDを共通に

### GPIO（マルチプレックス）
- セグメントライン（a..g, dp）を全桁で共通接続（8本）
- 各桁のコモン（digit select）を個別トランジスタで駆動（16本）
- トランジスタのベースに現在制限抵抗、コモン側に電源を正しく接続
- スキャン周波数（refresh_hz）を 500–1000Hz 程度に設定

## 推奨 `config.json` フィールド
- `panel`:
  - `cols`: 8
  - `rows`: 2
  - `digits`: 16（自動計算でも可）
  - `orientation`: 0 | 90 | 180 | 270
- `driver`:
  - `type`: "max7219" | "ht16k33" | "gpio"
  - ドライバ固有の設定項目（下記サンプル参照）
- `segment_map`（オプション）: セグメントビットのマップ
- `scan_order`: "row-major"（または "col-major"）

## セグメントビットマップ（推奨）
- 1バイト（8ビット）で1桁を表現する方式を推奨
  - bit0 = a
  - bit1 = b
  - bit2 = c
  - bit3 = d
  - bit4 = e
  - bit5 = f
  - bit6 = g
  - bit7 = dp

- 例: 数字 0 = a+b+c+d+e+f -> 0b00111111 (0x3F)

## 桁インデックス割当（row-major）
- 上段（row 0）: index 0..7（左→右）
- 下段（row 1）: index 8..15（左→右）

上段: [0] [1] [2] [3] [4] [5] [6] [7]

下段: [8] [9] [10] [11] [12] [13] [14] [15]

## テスト手順（簡易）
1. ドライバ接続確認
   - MAX7219: SPI が通っているか確認（`spidev` 等）
   - HT16K33: `i2cdetect -y 1` でアドレス確認
2. エミュレータで表示確認
   - リポジトリに含まれる `7seg_emulator.py` を使って表示を確認できます。
   - 例:
```bash
python3 7seg_emulator.py --config config.json
```
  （スクリプトの引数は実装に依存します。`--help` を参照してください）
3. 実機での段階的チェック
   - 全セグ点灯パターン（0xFF）を各桁に出力し、配線・短絡の確認
   - 各桁に 0..9 の数字を順に表示させる

## トラブルシューティング
- 片桁だけ点灯しない: 桁選択（コモン）配線やトランジスタを確認
- 全桁同一パターン: セグメント共通配線の短絡を疑う
- ちらつき・点滅: `refresh_hz` を上げる／ソフトウェア側でのスキャンタイミングを見直す
- 文字順がずれる: `orientation` や `scan_order`、`segment_map` を確認

## 物理配線と I2C（Raspberry Pi 向け注意点）
以下は `8x2` を TCA/エクスパンダなしで **直接 I2C モジュール（アドレス 0x70）** として接続する場合の注意点と配線例です。

- 電源: モジュールの VCC が **3.3V** か **5V** かを確認してください。Raspberry Pi の I2C は 3.3V ロジックなので、モジュールが5Vロジックならレベルシフタを必須で使用してください。
- SDA/SCL: モジュールの `SDA` を Pi の `SDA`、`SCL` を Pi の `SCL` に接続します（Raspberry Pi の GPIO ピンはモデルにより異なりますが、一般的にはピン3=SDA, ピン5=SCL）。
- GND: Pi とモジュールの GND を必ず共通接続してください。
- プルアップ: 多くの I2C モジュールは基板上にプルアップ抵抗を搭載しています。ない場合は 3.3V 側に 2.2k–10kΩ のプルアップ抵抗を追加してください。
- アドレス確認: 接続後に Pi で `i2cdetect -y 1` を実行して `0x70` が見えることを確認してください。

配線例（Raspberry Pi GPIO ヘッダ、3.3V ロジック）:

- Pi 3.3V (pin 1) -> モジュール VCC (if module requires 3.3V)
- Pi GND (pin 6) -> モジュール GND
- Pi SDA (pin 3, GPIO2) -> モジュール SDA
- Pi SCL (pin 5, GPIO3) -> モジュール SCL

チェックリスト（トラブルシュート順）:
- `i2cdetect -y 1` で `0x70` が表示されない場合:
  - 配線の向き（SDA/SCL/GND/VCC）を再確認
  - 電源電圧がモジュール仕様に合っているか確認
  - 別の I2C バス（`/dev/i2c-0` など）を試す（Pi の機種や設定でバス番号が異なることがある）
  - モジュールに 3.3V プルアップがない場合は手動で追加
- 初期化エラーが出る場合（`initialize_displays` が失敗）:
  - `config.json` の該当エントリに `"buses": { "0": { ... } }` が正しく設定されているか
  - モジュールアドレスを 0x70 として `"0x70"` で指定しているか（JSON 内は文字列で16進表記）

実行例（エミュレータではなく実機で動かす）:

